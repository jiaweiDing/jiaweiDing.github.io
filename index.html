OC与JS交互

##1.准备工作

  • 准备好html文件

  • 里面可能含有各种css、js文件，但主要的是html文件

  • 先把整个文件夹导入到工程中

  • 创建webView(一定把用户交互打开[self.webView setUserInteractionEnabled:YES];)

  • 读取html文件

[self.webView loadRequest:[NSURLRequest requestWithURL:[[NSBundle mainBundle] URLForResource:@”test” withExtension:@”html”]]];

  • 这里有个需要注意的地方

html源文件里面的路径需要更改成一级路径比如./septo.min.js

原本是按照本地路径
<link rel="stylesheet" href="../css/font/iconfont.css"/> <link rel="stylesheet" href="../css/app.css"/> <script src="../js/zepto.min.js"></script> <script src="../js/echarts-all.js"></script>

需要更改成<link rel="stylesheet" href="./iconfont.css"/> <link rel="stylesheet" href="./app.css"/> <script src="./zepto.min.js"></script> <script src="./echarts-all.js"></script>才能正常读取

##2.交互

###1.设值

  • 假如设值调用的方法是setnum(),传的参数是{num:%ld,last:%ld,middle:%ld,hight:%ld},那么调用的方法是setnum({num:%ld,last:%ld,middle:%ld,hight:%ld})

  • 设值需要用到代理方法

  • 遵循UIWebViewDelegate代理协议

  • 在代理方法webViewDidFinishLoad:中调取系统的设值方法

NSString *str = [NSString stringWithFormat:@”setnum({num:%ld,last:%ld,middle:%ld,hight:%ld},num,last,middle,hight)”]; #拼接调用JS的字符串

[self.webView stringByEvaluatingJavaScriptFromString:str]; #对webview设值

  • 准确无误后就能看到效果

  • 注：一般html请求都是用JSON数据,可直接拼字符串

###2.点击回调

  • 需要用到系统JS框架,在Link Binary With Libraries中导入JavaScriptCore.framework库，并包含<JavaScriptCore/JavaScriptCore.h>头文件

  • 回调使用的是block

  • 假设点击回调的方法名叫request_data,回调的参数有year,month,dic

  • 步骤：

JSContext *context=[webView valueForKeyPath:@”documentView.webView.mainFrame.javaScriptContext”]; #创建JSContext对象（此处通过当前 webView的键获取到jscontext）

context[@”request_data”] = ^() { #请求回调的数据
NSArray arr = [JSContext currentArguments]; #数组里就是回调的JSON数据
JSValue jsv = arr[0]; #取到第一个参数year
year = jsv.toString; #将JSON数据转成string
}

  • 如果需要点击回调后重新设置，可用上文提到的方法或者:

NSString *str = [NSString stringWithFormat:@”setnum({num:%ld,last:%ld,middle:%ld,hight:%ld},num,last,middle,hight)”]; #拼接调用JS的字符串

[context evaluatrScript:str]; #给webView设值

这样就OK了。

Hexo+Github搭建博客

无意中看到Hexo+Github搭建博客的过程，自己试了一下，搞定。所以分享出来给大家看看。

  • 抱着学习的态度研究了一番，凌晨2点，over
  • 此文主要为Mac版，windows也差不多

1.准备工作

1.1 安装Homebrew

  • Homebrew安装

curl -LsSf http://github.com/mxcl/homebrew/tarball/master | sudo tar xvz -C/usr/local –strip 1

  • 检查brew是否安装成功

brew -v

如果有
Homebrew 0.9.5 (no git repository)
这样的提示代表Homebrew安装成功，可以继续下一步。

1.2 安装Git和Node

  • 使用Homebrew安装

brew install git

brew install node

1.3 Hexo的安装与设置

npm，Node和Git都安装好了就可以开始安装Hexo了

npm install hexo -g #-g表示全局安装

  • 可能出现的问题

    1. 没有权限，命令前加上sudo即可
    2. 前提需要安装Xcode

  • 测试是否安装成功

hexo -v

  • 初始化博客

mkdir hexoblog # 新建 博客文件夹

cd hexoblog # 切换到 博客文件夹

hexo init # 初始化

sudo npm install # 安装依赖

hexo server #启动服务

启动服务出现INFO Hexo is running at http://0.0.0.0:4000/. Press Ctrl+C to stop.或者INFO Hexo is running at http://localhost:4000/. Press Ctrl+C to stop.代表服务已启动。

用浏览器打开http://localhost:4000/会出现一个本地的hexo网页，到这时候我们的本地网页已经建立完毕。

2.部署到GitHub

2.1 GitHub设置

  • 注册GitHub

  • creat repostory 取个名字 #格式一般为：[GitHub ID].github.io 如ID为jiaweiding，则名字为：jiaweiding.github.io

2.2 配置_config.yml

cd hexoblog

vi _config.yml

找到最后几行，更改内容：

deploy:

type: git      

repository: https://github.com/jiaweiding/jiaweiding.github.io.git

branch: master

冒号后一定要加空格

2.3 部署到GitHub

hexo clean

hexo g

hexo d #部署到GitHub

执行上述命令后，产生如下提示：

Username for ‘https://github.com‘: # 输入用户名.

Password for ‘https://xxx@github.com‘: # 输入用户密码.

INFO Deploy done: git # 提示部署成功.

  • 错误提示：

ERROR Deployer not found: git,执行以下命令:

npm install hexo-deployer-git –save #更改主题提示错误也可执行该句代码解决

  • 上一步完成后再次执行:

hexo d

2.4 访问博客

  • 以上步骤无误，访问博客地址：http://jiaweiding.github.io/

3.什么？你说主题丑了？

点进去看看hexo主题，肯定有你想要的。

  • 安装主题(pacman为例)

git clone https://github.com/A-limon/pacman.git themes/pacman

  • 启动主题

修改config.yml中theme属性为pacman即可。

  • 更新主题

cd themes/pacman

git pull

注：请先备份你的_config.yml文件后再升级

4.添加博文

hexo new “postName” #新建博文，双引号内是博文题目

博文会自动生成在博客目录下source/_posts/postName.md

编辑postName.md后：

hexo clean

hexo g

hexo d

这样就同步更新了博客里的文章
